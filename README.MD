# 🏆 GitHub Rozetleri ve Başarımları Rehberi (GitHub Achievements)

Bu rehber, GitHub profilinizde sergileyebileceğiniz tüm başarı rozetlerinin (achievements) ve öne çıkan rozetlerin (highlights) ne anlama geldiğini ve bunları nasıl kazanabileceğinizi açıklamaktadır.

GitHub başarımları, açık kaynak projelere yaptığınız katkıları ve platformdaki etkileşiminizi kutlamak için verilen dijital rozetlerdir. Rozetlerinizi profil ayarlarınızdan gizleyebilir veya herkese açık hale getirebilirsiniz.

---

## 🏅 Başarım Rozetleri (Achievements) ve Nasıl Alınırlar?

Bu rozetleri, GitHub üzerindeki günlük aktivitelerinizle (Pull Request açmak, cevap yazmak vb.) kazanabilirsiniz.

### 🌟 Starstruck
* **Nasıl Alınır:** Oluşturduğunuz bir deponun (repository) belirli sayıda yıldız (star) alması gerekir.
* **Seviyeler:**
  * **Varsayılan:** 16 Yıldız
  * **Bronz:** 128 Yıldız
  * **Gümüş:** 512 Yıldız
  * **Altın:** 4096 Yıldız

### ⚡ Quickdraw
* **Nasıl Alınır:** Açtığınız bir Issue (Sorun) veya Pull Request'i (PR), açıldıktan sonraki **5 dakika içinde** kapatırsanız kazanırsınız. Sadece 1 seviyesi vardır.

### 🤝 Pair Extraordinaire
* **Nasıl Alınır:** Başarıyla birleştirilmiş (merged) bir Pull Request'te "ortak yazar" (co-author) olarak etiketlenmeniz gerekir. (Commit mesajlarına `Co-authored-by:` eklenerek yapılır).
* **Seviyeler:**
  * **Varsayılan:** 1 PR
  * **Bronz:** 10 PR
  * **Gümüş:** 24 PR
  * **Altın:** 48 PR

### 🦈 Pull Shark
* **Nasıl Alınır:** Açtığınız Pull Request'lerin başarıyla ana projeye birleştirilmesi (merged) gerekir.
* **Seviyeler:**
  * **Varsayılan:** 2 PR
  * **Bronz:** 16 PR
  * **Gümüş:** 128 PR
  * **Altın:** 1024 PR

### 🧠 Galaxy Brain
* **Nasıl Alınır:** GitHub Discussions (Tartışmalar) kısmında verdiğiniz cevapların, soruyu soran kişi tarafından "Kabul Edilen Cevap" (Accepted Answer) olarak işaretlenmesi gerekir.
* **Seviyeler:**
  * **Varsayılan:** 2 Kabul Edilen Cevap
  * **Bronz:** 8 Kabul Edilen Cevap
  * **Gümüş:** 16 Kabul Edilen Cevap
  * **Altın:** 32 Kabul Edilen Cevap

### 🚀 YOLO
* **Nasıl Alınır:** Açtığınız bir Pull Request'i, başka birinden hiçbir kod incelemesi (code review) almadan doğrudan birleştirirseniz (merge) bu rozeti kazanırsınız. Sadece 1 seviyesi vardır.

### 💖 Public Sponsor
* **Nasıl Alınır:** GitHub Sponsors programı aracılığıyla açık kaynaklı bir geliştiriciye veya projeye sponsor (maddi destek) olduğunuzda kazanırsınız.

*(Not: "Heart On Your Sleeve" ve "Open Sourcerer" gibi bazı rozetler GitHub tarafından test edilmektedir ve her zaman kazanılamayabilir).*

## ✨ Highlights Badges

Bu rozetler, GitHub platformundaki özel statüleri ve katılımları temsil eder ve profilinizin sol alt köşesinde listelenir.

* **Pro:** GitHub Pro kullanıcısıysanız veya öğrenci paketiniz (GitHub Student Developer Pack) varsa bu rozeti alırsınız.
* **Developer Program Member:** GitHub Geliştirici Programı'na (Developer Program) kayıtlı bir üyeyseniz verilir.
* **Security Bug Bounty Hunter:** GitHub'ın kendi sistemlerindeki güvenlik açıklarını bularak "Bug Bounty" programına katkıda bulunanlara verilir.
* **GitHub Campus Expert:** Öğrenci topluluklarına liderlik eden, "GitHub Campus Program" katılımcılarına verilir.
* **Security Advisory Credit:** GitHub Güvenlik Danışmanlığı Veritabanı'na (Advisory Database) kabul edilmiş bir güvenlik uyarısı gönderdiğinizde alırsınız.
* **GitHub Star:** GitHub topluluğuna büyük katkı sağlayan ve ilham veren, GitHub tarafından özel olarak seçilmiş kişilere verilen çok prestijli bir rozettir.
